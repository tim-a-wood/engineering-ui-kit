# Component Standards

## Purpose

Orient authors to reusable component standards and define folder-level ownership.

## Scope

This directory README orients readers to authored standards in this directory. The ownership boundary below is operative; the sibling documents are the authored standards as of package 0.4.0.

## Source Inputs

- `MOCK-*`: approved hi-fi mockups used as app-specific visual calibration references, not exhaustive standards.
- `engineering-ui-kit-prd-v7-final-contracts-incorporated.md`.
- `engineering-ui-kit-gap-analysis-v4-final-contracts.md`.
- `engineering-ui-kit-public-reuse-research-report-v3-dark-first-hi-fi-aligned.md`.
- `engineering-ui-kit-high-level-delivery-plan-v0.1.md`.
- `engineering-ui-kit-phase-1-implementation-spec-v0.1.md`.

## Owns

Component inventory, reusable component behavior, categories, states, variants, accessibility notes, and component manifest alignment.

## Does Not Own

Page composition, app routing, or global architecture decisions.

## Required Future Content

A map from each component document to the component categories it owns, plus alignment with `component-manifest.json`.

## Traceability Notes

Use `CMP-*` IDs for component rules and reference `TOK-*`, `FND-*`, `LAY-*`, and `VAL-*` IDs where appropriate.

## Open Decisions

No Phase 1 blocking decisions. Content will be expanded in the phase identified below.
