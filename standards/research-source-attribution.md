# Research Source Attribution

## Phase 2 posture

Public sources informed the Engineering UI Kit through taxonomy, accessibility posture, documentation structure, and implementation-primitive research. Phase 2 does not embed third-party source text, icons, images, code, or branded visual identity.

The package is not a visual identity fork or branded derivative of any public design system.

## Source categories

| Source category | Allowed Phase 2 use | Prohibited Phase 2 use |
|---|---|---|
| Public design systems | Component taxonomy, documentation structure, accessibility expectations, and naming comparisons. | Copying branded visual identity, proprietary examples, icons, or long source text. |
| Accessibility references | Keyboard, ARIA, focus, contrast, and interaction posture. | Treating examples as complete Engineering UI Kit specs without project adaptation. |
| Component primitives | Evaluating suitable implementation primitives for later phases. | Shipping copied implementation code in this standards package. |
| Charting references | Chart taxonomy, interaction expectations, and accessibility considerations. | Copying branded chart themes or vendor-specific code. |
| Engineering UI examples | Understanding artifact, traceability, log, and diff workflows. | Claiming external product patterns as approved company standards. |

## Attribution model

Attribution is handled by conservative source-category notes and the component reference map. Later phases may add formal third-party notices if implementation code, icons, fonts, assets, or copied text are introduced.
